# Туман — бот отчётов + owner page

Готовый MVP под твою схему:

- сотрудники кидают ежедневный отчёт в Telegram-группу
- бот проверяет формат сообщения
- если есть ошибка, бот отвечает, что нужно исправить
- если всё правильно, данные сохраняются в PostgreSQL
- владелец открывает веб-страницу, видит месяц, дополняет редкие расходы вручную
- владелец скачивает Excel-файл по шаблону

## Что умеет MVP

### Telegram bot
- принимает текстовые отчёты из группы
- валидирует поля кассы
- валидирует сумму расходов
- если не понял конкретную строку, просит исправить именно её
- если не распознал статью расхода, предлагает выбрать категорию прямо в Telegram
- принимает от сотрудников только категории:
  - закуп
  - зп
  - доставка
  - сервис
  - чай
  - аптека
- пишет в ответ понятную ошибку, если формат неверный
- при успешной валидации обновляет отчёт за день

### Owner page
- вход владельца по паролю
- просмотр месяца
- таблица кассы по дням
- таблица расходов по категориям
- просмотр исходных сообщений бота
- ручное добавление расходов владельцем по всем категориям
- ручное редактирование и удаление расходов
- ручное исправление кассы за выбранный день
- экспорт в `.xlsx`

## Формат сообщения для сотрудников

```text
30.03
Наличные: 4060
Рубли: 16520
Банковские карты: 125736
Яндекс доставка: 1280
Нет монет: 0
Общая: 147596

Расходы:
1100 Арине (зп)
229 вода 5л (закуп)
250 дост (доставка)
4400 % (сервис)
180 чай липтон (чай)

Итого расход: 6159
Итог: 9501
```

`Итог` = осталось наличных.

## Как бот теперь помогает исправлять ошибки

- если в строке кассы или итоге некорректное число, бот отвечает на сообщение и просит прислать одну исправленную строку
- если в строке расхода непонятна статья, бот показывает эту строку и просит выбрать категорию кнопкой
- после каждого уточнения бот заново проверяет весь отчёт и либо задаёт следующий вопрос, либо сохраняет отчёт
- если удобнее, сотрудник всё так же может просто отредактировать исходное сообщение целиком

## Структура проекта

- `src/server.js` — express app + API + подключение Telegram webhook
- `src/telegram.js` — логика бота
- `src/parser.js` — парсер и валидация сообщений
- `src/reports.js` — работа с отчётами в базе
- `src/excel.js` — выгрузка Excel по шаблону
- `sql/schema.sql` — схема PostgreSQL
- `public/` — owner page
- `templates/tuman-march-2026.xlsx` — базовый шаблон Excel

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни:

- `DATABASE_URL` — PostgreSQL
- `OWNER_USERNAME` — внутреннее имя владельца для сессии, можно оставить `owner`
- `OWNER_PASSWORD` — пароль владельца
- `JWT_SECRET` — секрет для owner page
- `TELEGRAM_BOT_TOKEN` — токен бота
- `TELEGRAM_ALLOWED_CHAT_ID` — id группы, где бот принимает отчёты
- `APP_BASE_URL` — публичный URL Railway для webhook

## Запуск локально

```bash
npm install
cp .env.example .env
npm run start
```

Сервис поднимется на `http://localhost:8080`.

## Railway

### Один сервис
Для MVP можно держать всё в одном сервисе Railway:

- Express API
- owner page
- Telegram webhook

### Что задать в Railway Variables
- `DATABASE_URL`
- `OWNER_USERNAME`
- `OWNER_PASSWORD`
- `JWT_SECRET`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_ALLOWED_CHAT_ID`
- `APP_BASE_URL`
- `DEFAULT_REPORT_YEAR=2026`
- `AUTO_APPLY_SCHEMA=true`

Если `OWNER_PASSWORD` или `JWT_SECRET` не заданы и остались дефолтными плейсхолдерами, вход владельца будет отключён с явной ошибкой настройки вместо молчаливого `401`.

## Важная логика сохранения

- бот обновляет только сотруднические строки расходов
- расходы владельца не стираются, если сотрудник прислал новый отчёт за день
- владелец добавляет редкие категории вручную через страницу

## Excel

В Excel добавляется колонка `осталось наличных` на листе `Касса`.

При экспорте сервис:
- подставляет кассу по дням
- подставляет расходы по категориям
- чинит дневные формулы на листе `Расход`
- сохраняет файл и отдаёт его на скачивание
